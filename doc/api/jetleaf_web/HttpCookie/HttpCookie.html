Represents an HTTP cookie as a name-value pair, consistent with the content of the "Cookie" request header.
For cookies sent in responses, see ResponseCookie, which includes additional attributes such as expires, maxAge, domain, path, and secure expected in the "Set-Cookie" response header.

This class ensures that a cookie always has a non-empty name and that the value defaults to an empty string if not provided.

Validation Rules

  • The cookie name must not be empty.
    (Throws IllegalArgumentException if the name is empty.)
  • The cookie value may be empty, but is never null.

Example

final cookie = HttpCookie("sessionId", "abc123");
print(cookie.getName());  // Output: sessionId
print(cookie.getValue()); // Output: abc123
