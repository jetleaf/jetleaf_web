Sets the context path for the http request.

The context path defines the base URI prefix under which all request mappings and handlers are registered. It allows multiple applications to be deployed on the same host and port without conflicting paths.

Parameters

  • contextPath: The new context path to set. Must begin with a / character (e.g., /api, /app). Use / to indicate the root context.

Example

server.setContextPath("/api");
// Controller mapped to "/users" will now be accessible at "/api/users"

Notes

  • This method should be called before the server starts accepting requests.
  • Setting a context path of / means no path prefix will be applied.
  • The configured context path is typically exposed via getContextPath and used by routing and view resolution components.

Implementation
