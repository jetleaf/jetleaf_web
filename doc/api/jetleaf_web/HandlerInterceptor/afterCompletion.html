afterCompletion abstract method

Future<void> afterCompletion(
  1. ServerHttpRequest request,
  2. ServerHttpResponse response,
  3. HandlerMethod handler
)

Defines a contract for intercepting requests before and after they are handled by a HandlerMethod.

Handler interceptors are invoked in the following order:

  1. preHandle — before the handler method executes. Returning false aborts further processing.
  2. postHandle — after the handler method executes, but before the response is committed.
  3. afterCompletion — after the request has been fully processed, useful for cleanup tasks.

Interceptors can be used for:

  • Logging and auditing
  • Authentication and authorization checks
  • Performance metrics
  • Modifying requests or responses globally

Example:

class LoggingInterceptor implements HandlerInterceptor {
  @override
  Future<bool> preHandle(ServerHttpRequest request, ServerHttpResponse response, HandlerMethod handler) async {
    print('Incoming request: ${request.getRequestURI()}');
    return true; // continue processing
  }

  @override
  Future<void> postHandle(ServerHttpRequest request, ServerHttpResponse response, HandlerMethod handler) async {
    print('Handler executed for: ${request.getRequestURI()}');
  }

  @override
  Future<void> afterCompletion(ServerHttpRequest request, ServerHttpResponse response, HandlerMethod handler) async {
    print('Request completed for: ${request.getRequestURI()}');
  }
}

Invoked after the request has been completely processed, regardless of whether an exception occurred or the handler executed successfully. Useful for resource cleanup, logging, or auditing.

Parameters

  • request: The original HTTP request.
  • response: The HTTP response sent to the client.
  • handler: The handler method that was invoked.
