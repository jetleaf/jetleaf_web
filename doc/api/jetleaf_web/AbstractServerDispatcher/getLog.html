Log getLog()

Returns the Log instance used by the server dispatcher and related components to record operational and diagnostic information.

The logger provides structured, level-based logging capabilities (e.g., debug, info, warn, error) throughout the server lifecycle â€” including request handling, startup, and shutdown events.

Typical Usage

final log = getLog();
log.info('Server started on port 8080');
log.debug('Dispatching request: ${request.uri}');

Responsibilities

  • Captures and reports runtime events during request processing.
  • Aids in tracing and debugging of middleware, routes, and handlers.
  • Provides context-aware messages during startup and shutdown phases.

See also

  • ServerDispatcher for request routing and execution.
  • ApplicationContext for centralized access to the logging subsystem.
