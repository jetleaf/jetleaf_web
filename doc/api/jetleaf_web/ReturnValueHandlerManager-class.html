ReturnValueHandlerManager

ReturnValueHandlerManager class abstract interface

Central coordination interface for managing and delegating method return value handling within the JetLeaf web framework.

The ReturnValueHandlerManager acts as a composite delegate that manages all registered ReturnValueHandler instances.
It determines how controller method return values — such as objects, primitives, Futures, or framework-specific types like PageView or ResponseEntity —
are transformed into HTTP responses.

Responsibilities

  • Delegates return value processing to the correct ReturnValueHandler
  • Maintains an ordered list of available handlers
  • Provides a unified lookup via findHandler
  • Implements the ReturnValueHandler contract itself, allowing this manager to be used transparently wherever a handler is expected

Typical Workflow

  1. A controller or route method is invoked by a HandlerAdapter.
  2. Its return value is passed to ReturnValueHandlerManager.handleReturnValue.
  3. The manager selects the most suitable ReturnValueHandler using findHandler.
  4. The selected handler writes or renders the final response.

Example

final handler = manager.findHandler(method, result, request);
if (handler != null) {
  await handler.handleReturnValue(result, method, request, response, handlerMethod);
}
